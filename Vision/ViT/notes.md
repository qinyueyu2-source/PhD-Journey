# ViT 核心问答速查笔记 (Notes)

### Q1：论文中“恒定隐向量维度 D，将块映射到 D 维”是什么意思？
- **核心机制**：Transformer 要求所有层的数据维度恒定为 \(D\)。
- **操作拆解**：
  1. **展平**：将 2D 图像块（如 \(16\times16\times3\)）拉直成一维向量（长度 \(P^2 \cdot C\)）。
  2. **线性投影**：通过一个**可训练的权重矩阵 \(\mathbf{E}\)**（尺寸 \( (P^2 \cdot C) \times D \)），强行将不同长度的向量“压缩”或“扩展”为统一的维度 \(D\)。
- **产物**：输出称为 **Patch Embeddings**（块嵌入），相当于 NLP 中的词向量。

---

### Q2：Transformer 编码器的核心骨架是什么？
- **宏观堆叠**：由 \(N\) 个完全相同的 Block 堆叠而成（ViT-Base 有 12 层）。
- **每层内部（两个核心房间）**：
  - **房间 1**：多头自注意力（MSA）。负责让每个令牌（Token）**看全局**，计算所有令牌间的相关性。
  - **房间 2**：前馈网络（MLP）。负责对特征进行非线性变换和精炼。
- **标配连接**：每个房间外都包裹着**残差连接（Add）**和**层归一化（Layer Norm）**，即 “Add & Norm”。
- **核心公式**：\(\text{Attention}(Q, K, V) = \text{softmax}(\frac{QK^T}{\sqrt{d_k}})V\)（Q 查询，K 键，V 值，通过点积计算全局相似度）。

---

### Q3：BERT 是什么？与 ViT 有何血缘关系？
- **定义**：BERT 是 NLP 中**只使用 Transformer 编码器（Encoder）**的预训练模型。
- **两大预训练任务**：掩码语言模型（MLM，完形填空）和下一句预测（NSP）。
- **ViT 抄袭 BERT 的设计**：
  1. **模型尺寸**：直接沿用 BERT-Base/Large 的层数、维度、头数。
  2. **`[CLS]` 令牌**：在序列开头加一个特殊令牌，用于汇聚全局信息做分类。
  3. **位置编码**：使用**可学习的 1D 位置嵌入**，而非原始 Transformer 的正弦波。

---

### Q4：BERT 和 ViT 的“预训练-微调”具体是怎么操作的？
- **外科手术四步走**：
  1. **拆卸旧头**：扔掉预训练时的旧分类头（如 BERT 的词汇预测层，ViT 的 1000/18000 类输出层）。
  2. **安装新头**：在 `[CLS]` 令牌的输出位置，接一个**随机初始化**的线性层（维度 \(D \times K\)，\(K\) 为新任务类别数）。
  3. **微调（Fine-tuning）**：将**整个骨干网络（Backbone）+ 新头**一起训练。**关键技巧**：骨干网络学习率极小（如 \(10^{-5}\)），防止冲毁预训练知识；新头学习率较大，让它从零快速学会映射。
  4. **效果**：仅需少量标注数据，模型就能达到极好效果（知识迁移）。

---

### Q5：论文说的“归纳偏置（Inductive Bias）”到底指什么？
- **定义**：模型被提前赋予的“先天预设规则”。
- **CNN（强偏置）**：天生具备**局部性**（只看 3x3 邻域）、**平移等变性**（滑来滑去）和**二维结构**。像“带着尺子的老工匠”，小数据下表现极稳。
- **ViT（极弱偏置）**：**没有 2D 结构先验**。只有 MLP 层局部，**自注意力层全局**（第一个 Patch 直接看所有 Patch）。像“没带任何工具的裸眼天才”，空间关系必须**全靠海量数据去悟**。
- **结论**：小数据时 CNN 赢，大数据时 ViT 赢。

---

### Q6：位置编码加在一维序列上，为什么论文说它是“随机”且“不携带 2D 信息”？
- **区分“顺序”与“空间几何”**：
  - **有顺序**：虽然展平成了索引 1, 2, 3..., 但自注意力机制本身**不区分顺序**，交换索引位置算出来的结果一样。
  - **不携带 2D 信息**：初始化时，索引 0（左上）和索引 14（第二行第一列）的位置向量数值是**随机乱数**。它们的数学相似度（点积）可能为 0，也可能比索引 0 和索引 100 还要小。模型不知道它们实际上在二维空间里是“上下邻居”。
- **“从头学习”**：模型必须靠看几百万张图，硬记住“索引 0”和“索引 14”经常同时组成物体边缘，从而通过梯度下降把这两个随机向量的内部数值调成相似的。

---

### Q7：具体是怎么“调”位置向量让“点积变大”的？点积不是 Q 和 K 算的吗？
- **逻辑链补全**（数学机理）：
  1. **输入组成**：输入向量 = Patch内容 + 位置编码。
  2. **生成 Q/K**：输入向量分别乘以权重矩阵 \(W_Q\) 和 \(W_K\)，生成该位置的 \(Q\) 和 \(K\)。
  3. **点积**：确实是 \(Q \cdot K\) 在算，但 \(Q\) 和 \(K\) 的数值里**包含了位置编码的贡献**。
- **反向传播的魔法（怎么调）**：
  - 损失函数（Loss）反向传播时，链式法则会对网络**所有可训练参数**求导。
  - **梯度会传回给位置编码**（\(\frac{\partial Loss}{\partial PosEmbedding}\) 不为零）。
  - 优化器（Adam）拿到梯度，把位置向量里的数字往减小 Loss 的方向微调。经过亿万次迭代，索引 0 和索引 14 的向量被调得让 \(Q \cdot K\) 点积变大（相似）。
- **索引对应关系**：一维索引对应的物理空间位置（如左上角）在预处理时是**绝对固定且唯一**的，模型只是刚开始不知道这个含义。

---

### 终极思维导图（一句话串起来）
ViT 是将图像切成块，通过线性投影变成统一维度的序列（Q1），塞进源自 BERT 的 Transformer 编码器（Q2/Q3）中。它先在大数据上预训练，微调时换头并小学习率精调（Q4）。与 CNN 自带先天几何本能（强归纳偏置）不同，ViT 的 2D 位置信息全靠随机初始化的位置编码在一维序列上通过反向传播“硬背”出来（Q5/Q6/Q7），因此它极度依赖大数据，但也因此获得了更强的全局建模上限。

Image
  ↓
Patch Embedding
  ↓
[CLS] + Position Embedding
  ↓
Transformer Encoder × N
  ↓
CLS Token
  ↓
MLP Head
  ↓
Prediction



> **Transformer 并不是必须学习 Position Embedding。**

它只是**选择了一种方法**。

## 我最后想送你一个博士阶段特别重要的思维

你刚才的问题其实是在问：

> **"为什么不用一个人为设计的规则，而要让模型自己学？"**

这个问题不只是 Position Embedding，而是**整个深度学习的发展主线**。

例如：

* HOG 特征 → CNN 自己学特征
* SIFT 描述子 → CNN 自己学描述子
* 手工设计 Anchor → DETR 自己学 Query
* 手工设计 Prompt → LLM 自己学 Prompt（Soft Prompt）
* 固定 Position → Learnable Position

你会发现，近十几年的趋势几乎都是：

> **如果某种表示很难靠人精确设计，而且不同任务需求不同，那么就把它变成可学习参数，让数据决定最优表示。**

所以，**Position Embedding 并不是在学习"坐标"，而是在学习"如何把位置信息编码到特征空间中，才能最好地完成当前任务"。**

# 但是，这里还有一个更大的误区

很多初学者会认为：

> **Attention 热力图 = 模型真正思考的过程。**

实际上并不一定。

近年来有不少论文（比如 *Attention is not Explanation*）指出：

**Attention 权重高，并不一定意味着它就是模型最终决策最重要的依据。**

原因是：

* 信息还会经过后续的 MLP。
* 有残差连接（Residual），信息可以绕过 Attention。
* 后面的层还会继续修改表示。

所以现在研究模型可解释性，通常不会只看 Attention Heatmap，而是会结合梯度分析、遮挡实验（Occlusion）、Attention Rollout 等多种方法一起分析。

这也是为什么你会发现，同一个模型，不同论文画出来的解释图可能不完全一样。

## 最后，我想把这一部分浓缩成一句你以后要一直记住的话

**Transformer 并不是在学习"耳朵、眼睛、鼻子"这些人类概念。**

它学习的是：

> **一种能够降低任务损失（Loss）的高维表示和关系函数。**

而"耳朵关注眼睛""背景关注主体"这些描述，都是**我们为了理解模型行为而赋予它的解释**，并不是模型内部真的存在这样的标签。
